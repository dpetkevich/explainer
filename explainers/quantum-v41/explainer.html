How Shor's Algorithm Goes From Millions of Qubits to Ten Thousand

RSA-2048 and 256-bit elliptic-curve keys protect almost all of today's internet traffic. Breaking them with Shor's algorithm was assumed to need a quantum computer with millions of physical qubits, decades away. This paper argues the real number is 10,000 to 26,000, a drop of two to three orders of magnitude. That drop isn't because the qubits themselves got better. The key idea: swap an error-correcting code that only talks to its nearest neighbors for one that can wire any two qubits together. That swap turns most of the huge qubit budget from wasted overhead into usable qubits that actually compute. You'll follow one specific 256-bit key from the first scene to the last, watching the machine needed to break it shrink from millions of qubits to ten thousand.

The basic building blocks

Start with the target — a 256-bit elliptic-curve key — and the two physical facts standing in the way: real qubits are noisy, and the standard fix for that noise, the surface code, is wasteful. This part ends at the limitation the rest of the paper attacks.

Shor's algorithm turns an unbreakable-looking key into a period-finding problem

This explainer follows one target: a 256-bit elliptic-curve key, the kind protecting most of today's secure web traffic and cryptocurrency wallets.

Physical qubits fail thousands of times more often than Shor's algorithm can tolerate

Running Shor's algorithm on that 256-bit key takes billions of quantum operations, and each one can go wrong.

Spreading one qubit's information across many qubits suppresses errors exponentially

Raw qubits can't reach 1-in-10-billion reliability alone, so redundancy does the work instead: spread one qubit's information across many.

The standard code only talks to its neighbors, so it wastes almost all its qubits

That single logical qubit is a surface-code patch, the field's default design — but its neighbor-only wiring is expensive at scale.

The new technology

The surface code wastes qubits because it only wires neighbors together. This part shows the paper's fix:
  • a code that wires qubits at a distance
  • hardware (moving atoms) that can physically build those wires
  • the surgery and zone architecture that turns both into a working machine

Most gates in a circuit are free to correct for; the Toffoli gate is not

The million-qubit tax above comes from brute-force error correction, but not every gate even costs the same to run in the first place.

Letting checks connect distant qubits packs far more logical qubits into the same block

The surface code's overhead and the costly Toffoli gates from the scene before both need reliable qubits. A nonlocal check code — called qLDPC — packs far more logical qubits into the same physical block.

Optical tweezers can physically move atoms, wiring any two qubits together on demand

The qLDPC block above needs qubits that aren't neighbors to talk; reconfigurable atom tweezers make that connection physically real.

A temporary ancilla reads out one logical operation without disturbing the rest of the data

Now that atoms can reach any qubit in the block, watch the actual move: an ancilla — a helper qubit attached just to take a measurement — touches one logical operator, the qubit pattern that encodes the answer.

Splitting the chip into memory, processor, resource, and operation zones keeps the ancilla small

Doing surgery on the whole 1,480-qubit memory block would be impractical, so only 10-148 qubits move at a time to meet a magic state.

Why it beats existing tech

With the new code and architecture built up, put it head-to-head against the surface-code baseline from part 1 on the same two keys, then see how spending more qubits buys back speed.

The same ECC-256 job that needed millions of qubits now needs about 10,000

Put side by side on the same two keys and the same reliability target, the new architecture needs a roughly 100-1,000x smaller machine.

Running Toffoli gates one at a time is qubit-cheap but slow; running them in parallel costs more qubits but is fast

The 13,255-qubit number above is the frugal option; spend roughly 10x more qubits and the same job finishes nearly 100x faster.

Why now

The numbers in part 3 aren't hypothetical — check them against neutral-atom hardware that already exists, then close the loop on what a 10-day break of a 256-bit key means today.

The atom count this scheme needs is already within a few times of today's demonstrated arrays

The chip design from parts 2 and 3 needs 9,739 to 102,000 atoms. Current demonstrated arrays already reach 6,100+ atoms, more than half of the 9,739-atom low end.

Ten thousand reliable, reconfigurable qubits — not millions — stand between today's hardware and a 10-day break of ECC-256

Every scene here fed one number: with 9,739-13,255 qubits, the ECC-256 key from the very first scene falls in about 10 days.
